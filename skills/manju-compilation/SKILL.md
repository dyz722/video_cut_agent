# 漫剧合集剪辑策略

目标: 从多集漫剧素材中剪出"合集"成片 (高能混剪 / 名场面合集 / 单线剧情连播),
横屏 16:9 (1920x1080) 为主, 平台要求竖屏时用 fit=pad 上下留黑 + 字幕填充。

## 一、先确定合集类型 (问用户或从任务描述推断)

1. **高能名场面合集**: 打斗/反转/告白等情绪峰值片段, 不要求剧情连贯
2. **单角色向合集**: 某角色全部高光, 按时间线排序
3. **剧情连播合集**: 关键剧情拼成"N分钟看完", 必须保剧情因果链

## 二、素材分析要点

漫剧对白密度低、画面信息重, **ASR 不够, 必须看画面**:

- 每集先 transcribe + detect_scenes
- 漫剧的 scenes.json 切变极多(分镜), 以"对白段落 + 场景簇"为选片单位
- 用 task 派 subagent 逐集分析, 要求回传:
  `[start-end] 场景描述 | 情绪强度(1-5) | 出场角色 | 是否名场面`
  subagent 用 watch_video 看无对白的高能段 (BGM 突变/打斗往往无台词)
- 静音但镜头切换密集的区间 = 打斗/高能候选, 优先 watch_video 确认

## 三、选集与排序

- **开场必须是全合集情绪最高的 5-8s** (最炸的画面前置当钩子), 然后接
  "正片从头开始" -- 这是合集类内容的标准结构
- 高能合集: 按情绪强度波浪排序 (5-3-4-2-5...), 避免连续平段
- 剧情连播: 保留因果句 (transcript 中含人名+动机的对白), 过场全删
- 单段时长 6-20s; 总片长: 混剪 60-120s, 连播 3-8min

## 四、封装规范

- 片段之间用 fadeblack 0.4s 转场 (漫剧画风差异大, 硬切很跳)
- 开头钩子段加 hook 字幕点题 (如 "全程高能" / "三分钟看完XX"),
  effect=pop_in
- 对白字幕: style=default, 跟 transcript 时间戳; 漫剧原片自带字幕时
  **不要再加对白字幕** (用 watch_video 确认原片是否有内嵌字幕)
- 旁白补叙: 剧情连播需要串场时用 tts 生成旁白,
  voice=longanhuan_v3, instruction="你现在说话的角色是一个旁白，你说话的情感是neutral。"
  旁白处原声 volume 降到 0.3, 旁白字幕用 caption 样式
- BGM: 高能混剪可整片压一条节奏 BGM (volume 0.3, ducking=true);
  剧情连播保留原声不加 BGM

## 五、质检红线

- 开场 8s 不是全片最高能段 -> 重排
- 剧情连播出现因果断裂 (看不懂为什么) -> 补旁白或补片段
- 转场处黑屏超过 0.8s -> 检查转场参数
- 原片字幕与新加字幕重叠 -> 去掉新字幕或改位置
